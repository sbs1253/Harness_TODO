## 핵심 개념

**빌더 팀(`00-*`)** 이 프로젝트를 분석하여 **타겟 팀(`01-*~`)** 을 자동 생성합니다.
세팅이 완료(`ONBOARDING.md` 생성)되면 빌더 팀은 자동으로 **'서포트 모드'**로 전환되어 대기합니다.

---

## 7단계 파이프라인

```
[1] 요구사항 수집  →  [2] 문서 설계  →  [3] 팀 구성 + MCP 추천
                                              ↓ (선택)
                              [4] MCP/Skills 설치 (사용자)
                                              ↓
                              [5] 재검증 + 팀 강화
                                              ↓
                        [6] 보안 주입  →  [7] QA 검증 → ✅ 완료
```

---

## 트리거 키워드

| 입력 | 동작 |
|------|------|
| `하네스 세팅 진행해` | Phase 1 시작 |
| `00-doc-architect 진행해` | Phase 2 |
| `00-team-architect 진행해` | Phase 3 |
| `MCP 재검증` | Phase 5 (MCP 설치 후) |
| `00-security-enforcer 진행해` | Phase 6 |
| `00-qa-gate 진행해` | Phase 7 |
| `재검증` / `QA 재검증` | QA 재검증 (언제든) |
| `기획 변경 반영` | PRD 수정 후 팀 업데이트 |

> [!WARNING]
> **`MCP 재검증`** 과 **`재검증`** 은 다릅니다.
> - `MCP 재검증` → 팀 강화 (00-team-architect)
> - `재검증` → QA 재검사 (00-qa-gate)

## 자주 쓰는 패턴

### 기술 스택 모를 때
INTAKE_FORM에 "추천해줘"라고 적으면 AI가 최적 스택을 제안합니다.

### MCP 설치 없이 빠르게 진행
Phase 3 완료 후 바로 `"00-security-enforcer 진행해"` 입력 (Phase 4, 5 건너뜀)

### 기획 변경 시
`docs/PRD.md` 수정 → `"기획 변경 반영"` → 승인 → `"00-security-enforcer 진행해"` → `"재검증"`

### 에러 발생 시
각 Phase는 독립적으로 재실행 가능합니다. Phase 3에서 실패해도 처음부터 다시 할 필요 없어요.
