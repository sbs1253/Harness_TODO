# 🛡️ 프로젝트 공통 보안 및 품질 표준 (RULES)

이 문서는 모든 에이전트와 개발자가 준수해야 할 공통 표준을 정의합니다.

## 1. 보안 표준 (Security)
- **자격 증명 보호**: 모든 API 키, DB 비밀번호, API 엔드포인트 등 민감 정보는 반드시 `.env.local`에 저장하며, 소스 코드에 하드코딩하지 않습니다.
- **명령어 제한**: 시스템을 파괴할 수 있는 명령어(`rm -rf /`, `drop table` 등)의 실행을 엄격히 금지합니다.
- **에이전트 보호**: `.agents/skills/00-*` 하위의 빌더 팀 설정 파일은 실무 에이전트가 수정할 수 없습니다.

## 2. 개발 표준 (Development)
- **TypeScript**: 
  - `any` 타입 사용을 금지합니다. 불가피한 경우 이유를 주석으로 명시하십시오.
  - 모든 컴포넌트와 함수의 Props, Return 타입을 명시적으로 정의합니다.
- **React/Vite**:
  - 컴포넌트는 작은 단위로 분리하여 재사용성을 높입니다.
  - 비즈니스 로직은 Custom Hooks로 분리하여 관리합니다.
- **Tailwind CSS**:
  - 인라인 스타일링을 지양하고 Tailwind 유틸리티 클래스를 우선 사용합니다.
  - 일관된 디자인을 위해 `tailwind.config.js` 또는 전역 CSS 설정을 준수합니다.

## 3. 검증 프로세스 (Validation)
- 모든 작업 완료 전 반드시 아래 명령어를 통해 자가 검증을 수행합니다.
  - `npm run build`: 빌드 에러 유무 확인
  - `npm run lint`: 코드 컨벤션 및 문법 오류 확인
- 작업 결과는 항상 `docs/PRD.md` 및 `docs/ROADMAP.md`의 목표와 대조하여 검증합니다.

## 4. 변경 관리 (Change Management)
- 모든 의미 있는 변경 사항은 `docs/CHANGELOG.md`에 기록합니다.
- 작업 시작 전 `docs/PLANNING.md`를 통해 실행 계획을 수립하고 사용자의 승인을 받습니다.
